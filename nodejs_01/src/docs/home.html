API Interface System

Internal documentation and monitoring endpoints for the Node.js microservice.

Available Endpoints

POST /users
Description: Register a new user in the system database.
Payload: Expects a JSON object containing name and email.
Example: { "name": "Utsab Adhikari", "email": "contact@utsabadhikari.me" }
GET /users
Description: Fetches a complete list of all registered users.
Returns: An array of user objects in JSON format.
GET /requests
Description: Provides Handled request logs.
Usage: Useful for debugging and monitoring request flow through the system.
GET /errors
Description: Displays error logs and diagnostics.
Data Tracked: Includes error messages, stack traces, and timestamps for troubleshooting.
